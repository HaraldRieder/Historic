4.5 WDIALOG on Atari TOS

The print dialog inits the GEM metafile path with "*.*", therefore selection of a file outside the current directory is possible only after change of the drive.

Further effects observed together with the last English ACS3 beta were:

  • Print dialog works on a ST Emulator / Windows 98 installation.
  • Works not on a Magic 6.1 / Falcon installation (pdlg_create returns always NULL). Behaviour is equal for MIDI File Printer and the example programs that come together with WDIALOG.
  • Crashes on a TOS 4.04 on Falcon installation (inside aes/pdlg_create). The WDIALOG examples do NOT crash.
  • Print dialog hangs on a ST Emulator / Windows 2000 installation.

If you get too many undesired effects together with WDIALOG, select "Use own dialogs" in the "Dialogs" menu.

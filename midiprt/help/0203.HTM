Mad Harry's MIDI File Printer

2.3 The Information Window

This windows shows information from and about the MIDI file of the one document window that was most recently on top.

When there is no current document window, e.g. no MIDI file at all is loaded, the values are drawn in a way indicating that they are not valid.

The values have the following meanings:

  • File size: size of the file on disk in KByte (Atari) or Byte (Windows).
  • Size in memory: the standard MIDI file format is not suitable for fast access. The MIDI file printer therefore uses an own format which consumes more RAM. Here you see how much RAM is consumed by the transformed file.
  • MIDI file type: I tested the MIDI File Printer with type 0 and 1 standard MIDI files. These are the most widely spread types (I never ever saw a type 2 file). Type 0 contains 1 single track. Type 1 may contain more than 1 track, there usually track 0 is reserved for timing information.
  • No. of tracks: number of tracks that have been found in the MIDI file. Note that in general not every track will contain note events.
  • Ticks per beat: time resolution of the MIDI standard file in case of non-SMTPE format.
  • SMPTE: SMPTE value. The SMPTE timing format is not supported by the MIDI File Printer!
  • Ticks per frame: time resolution in case of SMPTE format.
  • Initial tempo: this value comes from the first tempo meta event found in the MIDI file.
  • Time signature: from the first time signature meta event.
  • Key signature: from the first key signature meta event. Major and minor are indicated by "maj" and "min", respectively.
  • Copyright Notice: the first copyright notice found in the MIDI file. On Atari TOS the copyright notice is shown in a separate window after pressing a button, which is disabled if no copyright notice was found.
